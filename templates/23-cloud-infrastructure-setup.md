# Cloud Infrastructure Setup

## Account and Organization
- [ ] Create cloud provider account (AWS, GCP, Azure)
- [ ] Set up organizational structure and accounts
- [ ] Enable MFA on root and all admin accounts
- [ ] Configure billing alerts and budgets
- [ ] Set up cost allocation tags
- [ ] Create IAM policies and roles

## Networking
- [ ] Design VPC or virtual network architecture
- [ ] Create subnets (public, private, data)
- [ ] Configure NAT gateway for private subnets
- [ ] Set up security groups and firewall rules
- [ ] Configure DNS and domain management
- [ ] Set up VPN or private connectivity if needed
- [ ] Enable VPC flow logs

## Compute
- [ ] Choose instance types based on workload
- [ ] Set up auto-scaling groups
- [ ] Configure launch templates or instance profiles
- [ ] Set up load balancers
- [ ] Configure health checks
- [ ] Set up bastion host or session manager for access

## Storage and Database
- [ ] Create object storage buckets with versioning
- [ ] Configure bucket policies and encryption
- [ ] Provision database instances (RDS, Cloud SQL, etc.)
- [ ] Set up automated backups and retention
- [ ] Configure read replicas if needed
- [ ] Set up database parameter groups

## Security
- [ ] Enable cloud trail and audit logging
- [ ] Set up security monitoring and alerting
- [ ] Configure encryption at rest and in transit
- [ ] Enable vulnerability scanning
- [ ] Set up secrets management (Vault, Secrets Manager)
- [ ] Review and restrict public access

## Monitoring and Observability
- [ ] Set up centralized logging
- [ ] Configure application performance monitoring
- [ ] Create dashboards for key metrics
- [ ] Set up alerts for critical thresholds
- [ ] Configure uptime checks
- [ ] Set up on-call notification routing

## CI/CD and Deployment
- [ ] Set up container registry
- [ ] Configure CI/CD pipelines
- [ ] Set up infrastructure as code (Terraform, CloudFormation)
- [ ] Create staging and production environments
- [ ] Configure deployment strategies (blue-green, rolling)
- [ ] Test full deployment pipeline end to end
