# Incident Response

## Detection and Triage
- [ ] Confirm the incident is real (not a false alarm)
- [ ] Assign incident severity level
- [ ] Identify affected systems and users
- [ ] Assign incident commander
- [ ] Open incident communication channel
- [ ] Start incident timeline log

## Communication
- [ ] Notify on-call team members
- [ ] Update status page for customers
- [ ] Send initial internal notification
- [ ] Identify stakeholders who need updates
- [ ] Set update cadence (every 15/30/60 minutes)

## Investigation
- [ ] Gather relevant logs and metrics
- [ ] Identify the root cause or contributing factors
- [ ] Determine blast radius and impact
- [ ] Check for related alerts or anomalies
- [ ] Document all findings in real time

## Mitigation
- [ ] Implement immediate fix or workaround
- [ ] Verify the fix resolves the issue
- [ ] Monitor for recurrence
- [ ] Roll back changes if fix doesn't work
- [ ] Confirm affected users are restored

## Resolution
- [ ] Verify all systems are healthy
- [ ] Update status page to resolved
- [ ] Send all-clear notification
- [ ] Confirm monitoring is catching normal patterns

## Post-Incident
- [ ] Schedule post-mortem meeting
- [ ] Write post-mortem document
  - [ ] Timeline of events
  - [ ] Root cause analysis
  - [ ] What went well
  - [ ] What could be improved
  - [ ] Action items with owners
- [ ] Share post-mortem with team
- [ ] Track remediation action items to completion
- [ ] Update runbooks based on learnings
