# Sprint Planning

## Pre-Planning
- [ ] Review previous sprint retrospective action items
- [ ] Groom and prioritize product backlog
- [ ] Ensure top stories have acceptance criteria
- [ ] Estimate stories that haven't been sized
- [ ] Check team capacity (vacations, holidays, on-call)
- [ ] Review any carryover items from last sprint

## Sprint Planning Meeting
- [ ] Review sprint goal with product owner
- [ ] Walk through prioritized backlog items
- [ ] Discuss technical approach for complex stories
- [ ] Identify dependencies and blockers
- [ ] Commit to sprint backlog based on capacity
- [ ] Assign owners to stories and tasks
- [ ] Confirm definition of done for each story

## Sprint Setup
- [ ] Create sprint board and move committed stories
- [ ] Set up any needed feature branches
- [ ] Update project tracking tool
- [ ] Schedule mid-sprint check-in
- [ ] Share sprint plan with stakeholders
- [ ] Identify risks and mitigation strategies

## During Sprint
- [ ] Hold daily standups
- [ ] Track progress against sprint burndown
- [ ] Address blockers immediately
- [ ] Coordinate with dependent teams
- [ ] Demo completed work to product owner
- [ ] Update story statuses in real time

## Sprint Close
- [ ] Verify all committed stories meet definition of done
- [ ] Prepare sprint review demo
- [ ] Conduct sprint review with stakeholders
- [ ] Hold sprint retrospective
  - [ ] What went well
  - [ ] What needs improvement
  - [ ] Action items for next sprint
- [ ] Update velocity metrics
